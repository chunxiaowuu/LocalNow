# Deployment (GitHub Pages + Render)

**English** | [中文](deployment.zh-CN.md)

Hosting architecture and the rate-limited deployment steps for LocalNow.

## Architecture

```
Browser
  │  static site
  ▼
GitHub Pages (frontend, Next.js static export)  ──fetch (Authorization: Bearer)──►  Render (backend FastAPI + LangGraph)
  https://chunxiaowuu.github.io/LocalNow                                            https://localnow-backend.onrender.com
                                                                                        │
                                                                                  Maps API / Meituan LongCat
```

- **Frontend**: GitHub Pages (static export, `basePath=/LocalNow`), auto-published by GitHub Actions.
- **Backend**: Render (Docker), holds all secrets, **LongCat-only** (`LLM_PROVIDER=longcat`, no other LLM key configured).
- **Login**: GitHub OAuth; the backend issues a token, the frontend stores it in localStorage and sends `Authorization: Bearer` (robust across sites, avoiding third-party-cookie blocking).

## Security / rate limiting

| | plan generations / day | modifications (replans) per plan |
|---|---|---|
| anonymous (by IP) | 1 | ≤ 3 |
| logged in (by user) | 3 | ≤ 9 |

Over the limit returns `429` with a message. Secrets live only in backend env vars and are never sent to the browser.

## Environment variables (set on the Render backend)

| Variable | Description | Example |
|---|---|---|
| `LLM_PROVIDER` | fixed to `longcat` (set by the blueprint) | `longcat` |
| `LONGCAT_API_KEY` | Meituan LongCat key | `ak_...` |
| `AMAP_API_KEY` | maps key | `...` |
| `SESSION_SECRET` | token-signing secret (auto-generated by the blueprint) | auto |
| `GITHUB_CLIENT_ID` / `GITHUB_CLIENT_SECRET` | GitHub OAuth App | |
| `ALLOWED_ORIGINS` | frontend origin (CORS) | `https://chunxiaowuu.github.io` |
| `FRONTEND_BASE` | redirect-back URL after login | `https://chunxiaowuu.github.io/LocalNow` |
| `OAUTH_REDIRECT_BASE` | backend public URL (OAuth callback) | `https://localnow-backend.onrender.com` |

> ⚠️ **Do not** set `GOOGLE_API_KEY` / `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` — production uses the LongCat key only.

Frontend build variable (GitHub repository Variable):

| Variable | Description | Example |
|---|---|---|
| `NEXT_PUBLIC_API_URL` | backend URL (inlined into the frontend at build time) | `https://localnow-backend.onrender.com` |

## Steps (in order)

**0. Merge the deploy branch into `main`** (`render.yaml` and the publish workflow must be on main). The first Pages deploy runs automatically and fails when variables are missing — fill them in below and re-run.

**1. Deploy the backend on Render**
1. render.com → **New + → Blueprint** → pick repo `chunxiaowuu/LocalNow` (reads `render.yaml`, creates `localnow-backend`).
2. Note its URL (e.g. `https://localnow-backend.onrender.com`).
3. In the service **Environment**, fill the secrets from the table above (`LONGCAT_API_KEY`, `AMAP_API_KEY`, `ALLOWED_ORIGINS`, `FRONTEND_BASE`, `OAUTH_REDIRECT_BASE`).

**2. Create a GitHub OAuth App** (GitHub → Settings → Developer settings → **OAuth Apps → New**)
- Homepage URL: `https://chunxiaowuu.github.io/LocalNow`
- **Authorization callback URL**: `https://localnow-backend.onrender.com/auth/github/callback`
- Copy the Client ID + generate a Secret, set them on Render as `GITHUB_CLIENT_ID` / `GITHUB_CLIENT_SECRET` (Render redeploys automatically).

**3. Deploy the frontend on GitHub Pages**
1. Repo **Settings → Pages** → Source = **GitHub Actions**.
2. Repo **Settings → Secrets and variables → Actions → Variables** → new `NEXT_PUBLIC_API_URL` = the backend Render URL.
3. **Actions** → run **"Deploy frontend (GitHub Pages)"** (re-run once the variable exists) → site at `https://chunxiaowuu.github.io/LocalNow`.

**4. Verify**
- Open the Pages site → top bar shows "plans remaining 1/1" (anonymous) → plan once → a second attempt is blocked with 429.
- Click **GitHub login** → authorize → redirected back, quota becomes 3/3 → 3 plans allowed.

## Free-tier notes (fine for a demo)

- The free Render instance sleeps after ~15 min idle → the first request cold-starts in ~30–60s.
- The rate-limit SQLite lives on ephemeral disk → **resets on redeploy/restart** (daily limits hold during normal uptime). For hard persistence, attach a Render Disk or an external Redis.

## Local development

Follow `README.md`: `LLM_PROVIDER` can be anything (e.g. `gemini`, faster); when the OAuth / rate-limit variables aren't set, the login button is hidden and limiting falls back to anonymous (1/day). One command: `docker compose up --build`.
